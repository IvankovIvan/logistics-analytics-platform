# ADR-002: Snapshot-модель загрузки данных

## Статус
Принято

## Контекст

Источник данных предоставляет CSV-файл,
содержащий полный snapshot shipment_line
за последние 3 месяца.

Строки могут:
- изменяться
- удаляться
- менять shipment_date

При этом в выгрузке отсутствуют поля updated_at и is_deleted.

## Решение

Использовать модель полной замены данных в окне (rolling window).

Алгоритм nightly-загрузки:

1. Загружаем CSV в staging.snapshot_import через COPY.
2. Выполняем базовую валидацию данных.
3. В транзакции:
   - DELETE из raw.shipment_line за последние 3 месяца
   - INSERT данных из staging
4. Пересчитываем mart за тот же период.

Rolling window = 3 месяца.

## Последствия

Плюсы:
- Корректная обработка удалений
- Корректная обработка изменения shipment_date
- Идемпотентность процесса
- Простая и устойчивая логика

Минусы:
- Невозможность incremental-загрузки
- Повышенная нагрузка при полной замене окна
