# ADR-004: Партиционирование raw.shipment_line по shipment_date

## Статус
Принято

## Контекст

LAP хранит полную историю shipment_line.
Ожидаемый объём данных — миллионы строк в год.

Nightly-процесс выполняет удаление и повторную загрузку
данных за последние 3 месяца.

Без партиционирования операция DELETE по окну
может стать дорогостоящей при росте объёма данных.

## Решение

Таблица raw.shipment_line будет партиционирована
по диапазону shipment_date (RANGE).

Партиции создаются по месяцу.

Пример:
- shipment_line_2026_01
- shipment_line_2026_02

Удаление данных за окно будет затрагивать
только соответствующие партиции.

## Последствия

Плюсы:
- Быстрое удаление данных за период
- Улучшенная производительность агрегаций
- Масштабируемость при росте объёма

Минусы:
- Необходимость управления созданием партиций
- Немного более сложная схема БД
