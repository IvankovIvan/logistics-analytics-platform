# ADR-005: Rolling window 3 месяца для nightly пересчёта

## Статус
Принято

## Контекст

Источник данных предоставляет полный snapshot
за последние 3 месяца.

В реальности изменения могут вноситься
задним числом в пределах 3 месяцев.

Редкие изменения в более глубокой истории возможны,
но считаются исключением.

## Решение

Nightly-процесс пересчитывает:

- текущий месяц
- предыдущий месяц
- месяц -2

Rolling window = 3 месяца.

Для редких корректировок в более глубокой истории
предусматривается ручной full-rebuild.

## Последствия

Плюсы:
- Баланс корректности и производительности
- Простая и предсказуемая модель
- Минимизация нагрузки

Минусы:
- Возможность редкого расхождения при изменениях глубже 3 месяцев
  до момента ручного rebuild
