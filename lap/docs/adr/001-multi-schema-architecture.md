# ADR-001: Многослойная архитектура схем (multi-schema)

## Статус
Принято

## Контекст

LAP проектируется как самостоятельная аналитическая платформа.
Система должна быть расширяемой, поддерживать многослойную аналитику
и обеспечивать изоляцию этапов обработки данных.

Рассматривались варианты:

1. Использование одной схемы (public)
2. Использование одной схемы lap
3. Использование нескольких схем (raw, staging, mart, reporting)

## Решение

Использовать многослойную архитектуру схем:

- raw — хранение исходных данных (как пришли)
- staging — подготовка и очистка данных
- mart — агрегированные витрины
- reporting — представления для отчётов (будущее расширение)

## Последствия

Плюсы:
- Чёткое разделение слоёв
- Упрощение сопровождения
- Повышенная безопасность (разделение прав доступа)
- Масштабируемость платформы

Минусы:
- Немного более сложная структура БД
