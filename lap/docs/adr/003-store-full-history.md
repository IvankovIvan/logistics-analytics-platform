# ADR-003: Хранение полной истории данных

## Статус
Принято

## Контекст

LAP проектируется как самостоятельная аналитическая платформа.
Система должна поддерживать:

- Пересчёт исторических данных
- Изменение методологии расчёта метрик
- Расширение аналитики (ABC, cohort, LTV и др.)
- Аудит и воспроизводимость расчётов

Рассматривались варианты:

1. Хранить только последние N лет
2. Хранить только rolling window
3. Хранить полную историю shipment_line

## Решение

LAP хранит полную историю shipment_line без ограничения по времени.

Удаление данных допускается только по отдельному регламенту,
а не автоматически по сроку хранения.

## Последствия

Плюсы:
- Независимость от источника данных
- Возможность полного пересчёта
- Поддержка расширенной аналитики
- Соответствие платформенному подходу

Минусы:
- Рост объёма базы данных со временем
- Необходимость партиционирования
